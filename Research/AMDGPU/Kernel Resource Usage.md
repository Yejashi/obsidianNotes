
| Metric                                    | Compile-Time Meaning                                                                | Runtime Behavior / Interpretation                                                                                                   |
| ----------------------------------------- | ----------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| **NumVGPRs**                              | Exact number of vector registers allocated **per thread** after register allocation | Fixed at runtime; directly limits max waves/SIMD → high values reduce occupancy; shows up as **VGPR-limited occupancy** in profiler |
| **NumSGPRs**                              | Exact number of scalar registers allocated **per wave**                             | Fixed; can limit occupancy if high; less commonly the bottleneck vs VGPRs but visible as **SGPR pressure** in profiler              |
| **NumAGPRs**                              | Number of accumulator registers used (matrix/WMMA ops)                              | Fixed; reduces available VGPR space → indirectly lowers occupancy; relevant for MFMA-heavy kernels                                  |
| **VGPR Spills**                           | Number of vector registers spilled to scratch memory due to register pressure       | Causes extra global memory traffic at runtime → increased latency, bandwidth pressure, and slowdown                                 |
| **SGPR Spills**                           | Scalar register spills to memory (rare)                                             | Similar to VGPR spills but typically lower impact; still introduces extra memory ops                                                |
| **Scratch Size**                          | Total per-thread scratch memory required (bytes) due to spills                      | Allocated in global memory; high values → increased memory traffic and latency; may reduce performance significantly                |
| **LDS Usage (Static)**                    | Amount of shared memory (LDS) statically allocated per workgroup                    | Limits number of concurrent workgroups per CU → reduces occupancy if large                                                          |
| **LDS Usage (Dynamic)**                   | Additional LDS requested at launch                                                  | Same as static: contributes to occupancy limit; depends on launch configuration                                                     |
| **Occupancy (compiler)**                  | Maximum theoretical waves/SIMD or % occupancy based on resource usage               | Upper bound only; runtime occupancy ≤ this value; affected by launch config and execution behavior                                  |
| **Wavefront Size**                        | Hardware-defined (usually 64 threads)                                               | Fixed; used to interpret per-thread vs per-wave resource usage                                                                      |
| **Max Waves per SIMD**                    | Hardware limit (e.g., 16 or 20 depending on arch)                                   | Absolute ceiling; compiler occupancy is capped by this                                                                              |
| **Workgroup Size (implied constraints)**  | Threads per block implied by kernel or launch bounds                                | Affects how many waves are formed → impacts achieved occupancy                                                                      |
| **Spill Loads/Stores (if reported)**      | Number of inserted memory ops due to spills                                         | Directly increases instruction count and memory pressure at runtime                                                                 |
| **Private Segment Size**                  | Memory per-thread in scratch (same as scratch size conceptually)                    | Allocated in global memory; contributes to memory traffic and cache pressure                                                        |
| **Kernel Argument Size**                  | Size of kernel argument buffer                                                      | Negligible for performance unless extremely large                                                                                   |
| **Barrier Count / Sync usage (implicit)** | Number of synchronization points inferred                                           | May reduce effective occupancy due to stalls, not a hard limit                                                                      |
